# Value Based Ecosystem Design

Een complete module voor het ontwerpen van publieke dienst ecosystemen voor het oplossen van maatschappelijke vraagstukken, gebaseerd op een waardengedreven oriëntatie, met een professionele twee-fasen aanpak van waardenonderzoek naar ecosysteem ontwerp.

## Overzicht

Deze module bied een professionele toolkit voor waardengedreven publieke innovatie:

### Kern Componenten
- **8 Gespecialiseerde Agents**: Hiërarchische orchestration van canvas naar ecosystem design
- **15 Geïntegreerde Workflows**: Twee-fasen waardengedreven ontwerpproces
- **3 Validatie Taken**: Consistentie en compleetheid bewaking
- **5 Knowledge Stores**: RAG-enabled domein-specifieke expertise

### Belangrijkste Innovaties
- **Interactive Mens-AI Partnership**: Bidirectional dialogue in plaats van eenrichtingsverkeer
- **Twee-Fasen Architectuur**: Professionele scheiding tussen ideation en implementatie
- **Iteratieve Cross-Phase Workflows**: Mogelijkheid om terug te gaan en aanpassingen door te voeren
- **Specialized Knowledge Management**: Hallucination-prevention door domein-specifieke RAG stores

## Installatie

```bash
bmad install value-ecosystem-design
```

## Componenten

### Agents (8 totaal)

#### Orchestration Layer (3 agents)
- **Master Orchestrator**: Overkoepelende regie, fase-overgangen, context bewaking
- **Canvas Orchestrator**: Fase 1 leiding - creativiteit en pragmatiek
- **Ecosystem Orchestrator**: Fase 2 leiding - wetenschappelijke onderbouwing

#### Specialist Agents (5 agents)
- **Legal Agent**: Juridische validatie en compliance
- **Ethical Agent**: Morele en waarden gebaseerde beoordeling
- **Stakeholder Analyst**: Belangenanalyse en engagement strategieën
- **Business Architect**: Organisatorische en proces ontwerp
- **Semanticist**: Betekenisvolheid en conceptuele integriteit

### Workflows (15 totaal)

#### Fase 1: Canvas Foundation (6 workflows)
- `canvas-foundation` - Basis Canvas invulling met stakeholder analyse
- `value-hierarchy` - Waarden hiërarchie ontwikkeling
- `stakeholder-analysis` - Gedetailleerde stakeholder mapping
- `validation-ethics` - Ethische validatie van Canvas
- `validation-legal` - Juridische haalbaarheidscheck
- `iterative-canvas-update` - Canvas aanpassingen tijdens iteratie

#### Fase 2: Ecosystem Design (6 workflows)
- `ecosystem-design` - Complete ecosystem architectuur
- `implementation-planning` - Uitvoeringsstrategie en roadmap
- `comprehensive-integration` - PVC-VSD integratie
- `correct-course` - Aanpassingen tijdens ontwikkeling
- `iterative-ecosystem-update` - Ecosystem evoluties
- `validation-ethics` - Uitgebreide ethische validatie

#### Cross-Phase Workflows (3 workflows)
- `comprehensive-pvc-vsd` - Volledige twee-fasen proces
- `create-story` - Gebruikersverhalen generatie
- `dev-story` - Story implementatie met validatie

### Tasks (3 totaal)
- `consistency-validator` - Cross-fase consistentie controle
- `completeness-checker` - Ontwerp compleetheid verificatie
- `context-registrar` - Continue context registratie

## Snelle Start

1. **Laad de master orchestrator:**
   ```
   @value-ecosystem-master
   ```

2. **Bekijk beschikbare commands:**
   ```
   *help
   ```

3. **Start je eerste waardengedreven project:**
   ```
   *canvas-foundation
   ```

## Module Structuur

```
value-ecosystem-design/
├── agents/                    # 8 agent definities
├── workflows/                 # 15 workflow folders
├── tasks/                     # 3 validatie taken
├── templates/                 # Herbruikbare templates
├── data/                      # Knowledge stores
│   ├── legal-knowledge/       # Juridische expertise
│   ├── ethical-frameworks/    # Ethische kaders
│   ├── stakeholder-analysis/  # Stakeholder modellen
│   ├── business-architecture/ # Organisatorische patronen
│   └── semantic-models/       # Betekenis modellen
├── config.yaml               # Module configuratie
└── README.md                 # Deze documentatie
```

## Configuratie

De module kan worden geconfigureerd in `bmad/value-ecosystem-design/config.yaml`

**Voornaamste instellingen:**
- `phase_management`: Fase-specifieke benaderingen
- `agent_orchestration`: Hierarchische orchestration parameters
- `knowledge_management`: RAG store configuraties
